# Notion × サイト連携 セットアップ手順

このガイドでは、Notionで管理するコンテンツ（日記・作品集・写真）をサイトに自動反映させるための設定手順を説明します。

---

## 1. Notion インテグレーションの作成

1. [Notion Developers](https://www.notion.so/my-integrations) にアクセス
2. 「新しいインテグレーション」をクリック
3. 以下を設定：
   - **名前**: `My Website` など任意の名前
   - **ワークスペース**: 自分のワークスペースを選択
   - **機能**: 「コンテンツを読み取る」にチェック
4. 「送信」をクリック
5. 表示される **シークレットキー**（`secret_xxxx...`）をコピーして控えておく

---

## 2. Notionデータベースの作成

合計4つのデータベースを作成します。

### 文章日記（diary）用データベース

| プロパティ名 | 種類 | 説明 |
|---|---|---|
| Name | タイトル | 記事タイトル（デフォルトで存在） |
| Date | 日付 | 投稿日 |
| Published | チェックボックス | チェックが入った記事だけサイトに表示 |
| Excerpt | テキスト | 一覧に表示する抜粋文（任意） |

記事本文はページの中に普通に書いてください。

### イラスト日記（comicdiary）用データベース

プロパティはdiaryと同じです。ページのカバー画像を設定するとサムネイルとして表示されます。

### 作品集（work）用データベース

| プロパティ名 | 種類 | 説明 |
|---|---|---|
| Name | タイトル | 作品タイトル（デフォルトで存在） |
| Date | 日付 | 制作日・公開日 |
| Published | チェックボックス | チェックが入った作品だけサイトに表示 |
| Category | セレクト | 「work」または「Life work」を選択 |
| Description | テキスト | 作品の説明文 |
| Tags | マルチセレクト | タグ（例: イラスト, デザイン, etc.） |
| URL | URL | 作品の外部リンク（任意） |

カバー画像を設定するとカードのサムネイルになります。
- **Category が「work」** → 「work（お仕事）」セクションに表示
- **Category が「Life work」** → 「Life work（くらしのデザイン）」セクションに表示

### 写真（photo）用データベース

| プロパティ名 | 種類 | 説明 |
|---|---|---|
| Name | タイトル | 写真のタイトル（任意） |
| Published | チェックボックス | チェックが入った写真だけサイトに表示 |
| Description | テキスト | キャプション（任意） |

カバー画像を設定してください。それがグリッドに表示される写真になります。
もしくは「Image」というファイルプロパティを追加して、そこに画像をアップロードしても表示されます。

---

## 3. データベースにインテグレーションを接続

作成した **各データベース（4つすべて）** で：

1. データベースページの右上「…」メニューをクリック
2. 「コネクト」→ 先ほど作成したインテグレーション名を選択
3. 「確認」をクリック

---

## 4. データベースIDの取得

各データベースのページをブラウザで開き、URLからIDを取得します：

```
https://www.notion.so/xxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=yyyyyy
                     ↑ これがデータベースID（32文字）
```

ハイフンは除いた32文字の英数字です。**4つのデータベース**それぞれのIDを控えてください。

## 5. Vercelへのデプロイ

### 5-1. GitHubリポジトリの準備

1. [GitHub](https://github.com) でリポジトリを作成
2. サイトのファイル一式をプッシュ：

```bash
cd mywebsite
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/あなたのユーザー名/リポジトリ名.git
git push -u origin main
```

### 5-2. Vercelでデプロイ

1. [Vercel](https://vercel.com) にGitHubアカウントでログイン
2. 「Add New Project」をクリック
3. 先ほどのGitHubリポジトリをインポート
4. 「Framework Preset」は `Other` を選択
5. 「Environment Variables」に以下を追加：

| 変数名 | 値 |
|---|---|
| `NOTION_API_KEY` | Notionのシークレットキー（`secret_xxxx...`） |
| `NOTION_DIARY_DB_ID` | 文章日記のデータベースID |
| `NOTION_COMIC_DB_ID` | イラスト日記のデータベースID |
| `NOTION_WORK_DB_ID` | 作品集のデータベースID |
| `NOTION_PHOTO_DB_ID` | 写真アーカイブのデータベースID |

6. 「Deploy」をクリック

---

## 6. 使い方

セットアップ完了後は：

1. Notionのデータベースに新しいページを作成
2. 必要なプロパティを入力
3. `Published` にチェックを入れる
4. → サイトに自動で反映されます

Notionでの変更はサイトにリアルタイムで反映されます（ページを開くたびにNotionから最新データを取得するため）。

---

## トラブルシューティング

- **記事が表示されない**: `Published` チェックボックスがオンになっているか確認
- **APIエラー**: Vercelの環境変数が正しく設定されているか確認
- **権限エラー**: データベースにインテグレーションが接続されているか確認
- **プロパティ名に注意**: `Date`, `Published`, `Excerpt`, `Category`, `Tags`, `Description`, `URL`, `Image` は英語で正確に入力してください
- **Workのセクション分けがされない**: Category プロパティで「work」か「Life work」を正しく設定してください
- **写真が表示されない**: カバー画像を設定するか、「Image」ファイルプロパティに画像をアップロードしてください
